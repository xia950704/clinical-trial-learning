# 小样本研究（100例）：判断框架与落地路径

> 提取日期：2026-06-29
> 核心目标：判断100例病例是否值得继续分析
> 关联笔记：[[多中心数据管理_变量字典先行]]、[[病案首页真实世界研究可行性判断]]

## 一、核心判断

**只有100例病例，并不等于"不能分析"；但也不等于"可以做完整模型"。**

真正需要判断的是：这100例要回答什么问题，结局事件有多少，候选变量有多少，最后准备把结果定位为描述、探索，还是正式验证。

> 最怕的不是样本少，而是把一个适合做初步估计的问题，硬写成因果证明、疗效结论或临床预测模型。

---

## 二、四步判断流程

### 第一步：先看研究问题——你到底想回答哪一种问题？

同样是100例，研究问题不同，价值完全不同：

| 研究问题 | 100例是否够用 |
|---------|-------------|
| 本中心某类患者的临床特征、治疗路径和短期结局如何 | ✅ 可形成有边界的描述性队列 |
| 某个指标是否独立影响死亡 | ⚠️ 看死亡事件有多少、混杂因素能否控制 |
| 建立一个能预测患者结局的模型 | ❌ 通常非常吃紧 |

### 第二步：再看事件数——100例里真正发生结局的有多少？

**事件数才决定天花板，不是总样本量。**

- 100例里有60例发生结局 vs. 100例里只有6例发生结局 → 完全不同的研究
- 过去"每个变量至少10个事件"的经验规则并非绝对界线，但提醒：事件太少时，多变量模型不稳定，置信区间会很宽
- 事件数少时，候选变量必须克制

### 第三步：判断研究类型，决定统计方法能走多远

| 研究类型 | 100例的初步判断 | 建议边界 |
|---------|----------------|---------|
| **描述性病例队列** | 可继续 | 重点报告人群、治疗流程、结局发生情况和置信区间；不把描述结果写成因果结论 |
| **预后因素探索** | 有条件继续 | 先限定1–3个临床预设变量，明确混杂因素；事件数少时少做或不做多变量模型 |
| **预测模型开发** | 多数不适合 | 100例通常难以支撑稳定建模，尤其不适合机器学习和大量变量筛选 |
| **疗效比较** | 谨慎判断 | 若无随机化、结局少或组间差异大，应优先写成探索性估计，不能下疗效定论 |
| **可行性/预研究** | 较适合 | 重点回答缺失率、随访率、事件率、变量可获得性，为扩样或正式研究提供依据 |

### 第四步：变量越多，越要先做"减法"

统计模型不是变量仓库。变量越多，缺失越复杂，比较次数越多，偶然显著的概率也越高。

**变量分层策略**：

| 层级 | 内容 | 用途 |
|------|------|------|
| 第一层 | 必须描述的人群特征 | 主分析必用 |
| 第二层 | 临床上有明确依据的核心暴露或指标 | 主分析必用 |
| 第三层 | 探索性补充变量 | 补充或附录呈现 |

---

## 三、分析前必问4个问题

1. 研究对象是否连续纳入？纳排标准是否能被别人复核？
2. 主要结局是否只有一个？结局发生时间和判定标准是否一致？
3. 候选变量是否来自临床问题，而不是来自"Excel里有什么"？
4. 结果准备写成描述、探索还是验证？不同定位决定不同表述边界。

---

## 四、落地路径：可分析性核查

对于100例左右的病例库，建议先完成"可分析性核查"，再进入统计。

| 核查内容 | 说明 |
|---------|------|
| 纳排表 | 哪一时间段、哪个病区或门诊、是否连续入组、排除了哪些病例、排除原因是什么 |
| 变量字典 | 每个变量的定义、单位、来源、时间窗口 |
| 结局事件表 | 主要结局发生多少例、未发生多少例、失访或无法判断多少例 |
| 缺失情况 | 各变量的缺失率、缺失模式 |
| 时间窗 | 随访起点、终点和删失规则 |
| 最小分析计划 | 主分析围绕预先提出的核心问题展开 |

---

## 五、落地路径：四步执行

| 步骤 | 核心任务 |
|------|---------|
| **01 先做病例流向** | 说明100例从哪里来，决定研究对象是否清楚，关系到选择偏倚的判断 |
| **02 把结局事件单独列出来** | 主要结局发生多少例、未发生多少例、失访或无法判断多少例。事件表比一堆P值更能帮助读者判断研究能走多远 |
| **03 主分析要克制，补充分析要透明** | 主分析只围绕预先提出的核心问题展开。变量数量应与事件数匹配。敏感性分析用于检验缺失处理、不同结局定义或不同时间窗下结果是否方向一致，但不应把敏感性分析写成新的主要发现 |
| **04 结果表达要把"不确定性"写出来** | 报告效应量和置信区间，比只强调P值更适合小样本。若置信区间很宽，应承认估计不精确，而不是用"趋势明显"替代证据 |

---

## 六、什么时候应该暂缓？

并不是所有100例都值得继续。以下情况应先补数据库，而不是直接跑统计：

- 主要结局没有统一定义
- 关键变量缺失严重
- 病例来源不清
- 研究问题需要长期随访但现有资料只有住院期信息
- 研究者想回答疗效或因果问题，但数据来自非随机、非连续、治疗选择高度受病情影响的回顾性资料

---

## 七、实用判断

> **100例病例值得继续分析的前提，不是"能不能跑出显著性"，而是能不能形成一个清楚、克制、可复核的研究问题。**

- 能回答描述问题，就不要包装成预测模型
- 只能做探索，就不要写成确认性结论

## 九、关键数字速查

| 指标 | 说明 |
|------|------|
| 核心判断 | 研究问题 + 事件数 + 候选变量数 + 研究定位 |
| 事件数规则 | "每个变量至少10个事件"是经验规则，非绝对界线 |
| 变量分层 | 第一层（人群特征）+ 第二层（核心暴露）+ 第三层（探索性） |
| 主分析原则 | 变量数量应与事件数匹配，围绕预先提出的核心问题展开 |
| 结果表达 | 效应量 + 95%置信区间 > 单纯P值 |

# 数据管理与医学监察 — 索引

> 基于 pyCoreGage 架构的医学数据管理插件，支持临床试验数据清洗、异常值检测、医学编码、安全性监测

---

## 📋 模块概览

| 模块 | 文件 | 大小 | 功能 |
|------|------|------|------|
| **插件入口** | medcage.py | 13,212 bytes | 主协调器，加载配置、初始化检查器、执行检查、聚合结果 |
| **时间序列检查** | temporal_checker.py | 23,637 bytes | 访视顺序、时间锚定、时间窗、缺失时间检查 |
| **医学编码检查** | coding_checker.py | 28,618 bytes | MedDRA、ICD-10、ATC、LOINC 编码检查 |
| **安全性检查** | safety_checker.py | 21,742 bytes | AE 分级、SAE 识别、AESI 检查、实验室异常检查 |
| **合规性管理** | compliance_manager.py | 25,652 bytes | 审计追踪、电子签名、数据完整性、权限管理 |
| **核心数据结构** | core_data.py | 11,473 bytes | CoreGageConfig、CoreGageState、CoreGageResult 等 |

**总计**：124,334 bytes（121.4 KB）

## 🏗️ 架构设计

### 整体架构

```
pyCoreGage-医学插件/
├── medcage.py              # 插件入口（主协调器）
├── temporal_checker.py     # 时间序列检查模块
├── coding_checker.py       # 医学编码检查模块
├── safety_checker.py       # 安全性检查模块
├── compliance_manager.py   # 合规性管理模块
├── core_data.py            # 核心数据结构
└── dictionaries/           # 医学词典目录
    ├── README.md           # 词典使用说明
    ├── meddra/             # MedDRA 词典
    ├── icd10/              # ICD-10 词典
    ├── atc/                # ATC 词典
    ├── loinc/              # LOINC 词典
    ├── snomed/             # SNOMED CT 词典
    └── custom/             # 自定义词典
```

### 核心数据结构

| 结构 | 说明 |
|------|------|
| **CoreGageConfig** | 配置数据类，含路径、规则、词典等配置 |
| **CoreGageState** | 运行态对象，贯穿整个检查流程 |
| **CoreGageResult** | 检查结果，含发现、统计、报告 |
| **CoreGageFinding** | 单个发现，含 ID、严重级别、描述、记录 |

### 检查流程

```
1. 初始化 MedCage(config_path)
2. 加载数据 load_data(data_path)
3. 运行检查 run()
   ├── 时间序列检查 (TemporalChecker)
   ├── 医学编码检查 (CodingChecker)
   ├── 安全性检查 (SafetyChecker)
   └── 合规性检查 (ComplianceManager)
4. 保存结果 save_results(output_path)
```

## 🔍 检查模块详解

### 1. 时间序列检查 (temporal_checker.py)

| 检查项 | 方法 | 逻辑 |
|--------|------|------|
| **访视顺序** | `_check_visit_order()` | 按日期排序访视，检查 SCREENING→BASELINE→DOSING→FOLLOWUP 逻辑顺序 |
| **时间锚定** | `_check_time_anchor()` | AE 起始时间必须 ≥ 首次给药时间 |
| **时间窗** | `_check_time_window()` | 检查给药前 24h 时间窗内是否存在合并用药 |
| **缺失时间** | `_check_missing_time()` | 检查必须访视缺失 + 关键数据集时间字段缺失率 |

**配置化**：`TemporalCheckConfig` 支持自定义字段名映射（VISITDT/AESTDTC/EXSTDTC/COSTDTC 等 CDISC 标准字段）、时间窗大小、必检访视列表。

---

### 2. 医学编码检查 (coding_checker.py)

| 检查项 | 函数 | 目标域 | 检查逻辑 |
|--------|------|--------|----------|
| **MedDRA** | `check_meddra()` | AE | LLT→PT→SOC 层级映射验证 + 8 位数字编码格式检查 |
| **ICD-10** | `check_icd10()` | DI/CM | 字母+2 数字格式 + 章节字母有效性 + 可选扩展码 |
| **ATC** | `check_atc()` | CM/EX | 字母+2 数字+字母+字母+2 数字格式 + 一级分类验证 |
| **LOINC** | `check_loinc()` | LB | 5 位数字-1 位数字格式验证 |

**参考数据**：内置常用编码字典（MedDRA 19 条、ICD-10 28 章节、ATC 14 分类、LOINC 17 项），支持外部 CSV 参考表覆盖。

---

### 3. 安全性检查 (safety_checker.py)

| 检查项 | 方法 | 逻辑 |
|--------|------|------|
| **AE 分级** | `check_ae_grading()` | CTCAE v5.0（1-5 级）分级检查 |
| **SAE 识别** | `check_sae_criteria()` | 严重性标准（死亡/住院/残疾）检查 |
| **AESI 检查** | `check_aesi()` | 特别关注事件（如 TdP、QTc）检查 |
| **实验室异常** | `check_lab_abnormality()` | 参考范围、分级标准检查 |

---

### 4. 合规性管理 (compliance_manager.py)

| 功能 | 方法 | 逻辑 |
|------|------|------|
| **审计追踪** | `audit_trail()` | 记录所有操作（用户、时间、动作） |
| **电子签名** | `electronic_signature()` | 21 CFR Part 11 合规电子签名 |
| **数据完整性** | `data_integrity_check()` | ALCOA+ 原则检查 |
| **权限管理** | `access_control()` | 角色权限、访问控制 |

## 🔧 使用方法

### 1. 安装 pyCoreGage

```bash
pip install pycoregage
```

### 2. 配置医学插件

```python
from medcage import MedCage

# 创建配置
config = {
    "rule_registry": "rule_registry_medical.xlsx",
    "trial_checks": "rules/trial/",
    "study_checks": "rules/study/",
    "inputs": "inputs/",
    "reports": "outputs/reports/",
    "feedback": "outputs/feedback/",
    "project_name": "clinical_trial_demo",
}

# 初始化医学插件
mc = MedCage(config)

# 加载数据
mc.load_data("clinical_data.csv")

# 运行检查
mc.run()

# 保存结果
mc.save_results("output/report.json")
```

### 3. 规则注册表

创建 `rule_registry_medical.xlsx`，包含以下 Sheet：

| Sheet | 说明 |
|-------|------|
| **Trial** | 试验级检查规则 |
| **Study** | 研究级检查规则 |

每行定义一条检查规则：

| 列 | 含义 |
|----|------|
| Category | 分类（如 "Temporal" / "Coding" / "Safety"） |
| ID | 唯一检查标识符 |
| Active | Yes/No — 控制是否启用 |
| Rule_Set | 脚本名（如 "Temporal" / "Coding" / "Safety"） |
| Description | 人类可读描述 |

## 🔄 与 IIT 工作流的集成

| IIT 阶段 | 医学插件模块 | 集成方式 |
|---------|-------------|---------|
| **阶段 1：数据获取** | MedCage.load_data() | 加载 EDC 导出数据 |
| **阶段 2：交叉验证** | TemporalChecker.check_missing_time() | 检查受试者完整性 |
| **阶段 3：数据清洗** | TemporalChecker + CodingChecker | 时间序列 + 编码检查 |
| **阶段 4：医学编码** | CodingChecker.check_meddra/icd10/atc/loinc() | 医学编码验证 |
| **阶段 5：分组确认** | TemporalChecker.check_visit_order() | 访视顺序验证 |
| **阶段 6：Table 1** | SafetyChecker.check_lab_abnormality() | 基线特征检查 |
| **阶段 7-9：统计分析** | SafetyChecker.check_ae_grading/sae/aesi() | 安全性分析 |
| **阶段 10：Protocol** | ComplianceManager.audit_trail() | 审计追踪 |

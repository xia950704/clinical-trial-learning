# 中期分析与组序贯设计

## 1. 中期分析的目的和时机

### 1.1 中期分析的定义

**中期分析（Interim Analysis）** 是在临床试验进行过程中，在最终分析之前对累积数据进行的一次或多次统计分析。中期分析允许研究者在试验结束前做出关键决策。

### 1.2 中期分析的目的

| 目的 | 说明 |
|------|------|
| **早期终止（有效性）** | 当疗效证据足够强时，提前结束试验，让对照组患者也能获得有效治疗 |
| **早期终止（无效性）** | 当疗效证据明显不足时，提前终止试验，避免资源浪费和患者暴露于无效治疗 |
| **安全性监测** | 监测不良反应信号，保护受试者安全 |
| **样本量重估** | 根据中期数据重新估计效应量和样本量需求 |
| **适应性调整** | 调整入组标准、剂量选择或终点定义 |

### 1.3 中期分析的时机

- **预定义时间点**：在试验方案中预先指定（如完成50%、75%随访时）
- **事件驱动**：达到预定数量的终点事件（如死亡、疾病进展）
- **时间驱动**：固定随访时间点
- **数据量驱动**：达到预定样本量或随访时长

### 1.4 中期分析的伦理考量

> **伦理原则**：当有充分证据表明治疗有效或无效时，继续试验可能违反伦理。中期分析是平衡科学严谨性和伦理责任的重要工具。

## 2. 组序贯设计原理

### 2.1 组序贯设计概述

**组序贯设计（Group Sequential Design, GSD）** 是一种允许多次中期分析的试验设计框架，通过调整显著性水平来控制总体I类错误率。

### 2.2 多重性问题

每次中期分析都是一次假设检验，多次检验会增加假阳性率（I类错误膨胀）：

$$
\alpha_{overall} = 1 - (1-\alpha)^k
$$

其中 $k$ 为分析次数。组序贯设计通过**边界调整**控制总体α。

### 2.3 经典组序贯边界

#### 2.3.1 O'Brien-Fleming 边界

**特点**：
- 早期边界非常严格（α极小），后期边界逐渐放宽
- 适用于希望保守对待早期结果的试验
- 对最终分析影响最小

**边界值示例**（2次分析，总体α=0.05）：

| 分析次数 | 边界值（Z） | 边界值（α） |
|----------|-------------|-------------|
| 第1次（中期） | 3.471 | 0.0005 |
| 第2次（最终） | 2.179 | 0.0146 |

**适用场景**：疗效证据需要非常强的支持，避免过早终止试验。

#### 2.3.2 Pocock 边界

**特点**：
- 每次分析使用相同的边界值
- 早期更容易终止试验
- 对最终分析的α消耗较多

**边界值示例**（2次分析，总体α=0.05）：

| 分析次数 | 边界值（Z） | 边界值（α） |
|----------|-------------|-------------|
| 第1次（中期） | 2.360 | 0.0091 |
| 第2次（最终） | 2.360 | 0.0091 |

**适用场景**：希望尽早获得结论，对早期终止持开放态度。

#### 2.3.3 Alpha Spending Function（α消耗函数）

**核心思想**：将α按信息时间（information fraction）分配，而非固定边界。

**常用消耗函数**：

| 函数 | 公式 | 特点 |
|------|------|------|
| **O'Brien-Fleming型** | $\alpha(t) = 2 - 2\Phi\left(\frac{z_{\alpha/2}}{\sqrt{t}}\right)$ | 早期保守，后期宽松 |
| **Pocock型** | $\alpha(t) = \alpha \cdot \ln(1 + (e-1)t)$ | 早期较宽松 |
| **线性** | $\alpha(t) = \alpha \cdot t$ | 均匀分配 |

其中 $t$ 为信息时间（0 < t ≤ 1），$\Phi$ 为标准正态分布函数。

**优势**：
- 允许灵活的中期分析时间
- 可根据实际入组进度调整分析计划
- 边界值自动适应信息量

### 2.4 边界比较

```
Z值
  ↑
  │     ┌───────────────────── O'Brien-Fleming
  │    /
  │   /
  │  /
  │ /
  │/───────────────────────── Pocock
  │
  └──────────────────────────→ 信息时间
```

## 3. 贝叶斯中期分析

### 3.1 贝叶斯框架的优势

| 方面 | 频率学派 | 贝叶斯学派 |
|------|----------|------------|
| 参数理解 | 固定未知 | 随机变量，有概率分布 |
| 先验信息 | 无法纳入 | 可通过先验分布纳入 |
| 中期决策 | 需调整α | 直接计算后验概率 |
| 解释性 | p值易误解 | 后验概率直观 |

### 3.2 贝叶斯中期分析的核心概念

#### 3.2.1 先验分布

$$
\theta \sim \text{Prior}(\theta)
$$

常用先验：
- **无信息先验**：$\theta \sim N(0, \sigma^2)$，$\sigma^2$ 很大
- **弱信息先验**：基于历史数据或专家意见
- **信息先验**：基于充分的先验证据

#### 3.2.2 后验分布

$$
p(\theta | \text{data}) \propto p(\text{data} | \theta) \cdot p(\theta)
$$

#### 3.2.3 决策规则

**有效性判定**：
$$
P(\theta > \delta_{\text{min}} | \text{data}) > \gamma_{\text{eff}}
$$

**无效性判定**：
$$
P(\theta > \delta_{\text{min}} | \text{data}) < \gamma_{\text{futility}}
$$

其中 $\delta_{\text{min}}$ 为最小临床有意义效应，$\gamma$ 为概率阈值。

### 3.3 贝叶斯中期分析的优缺点

**优点**：
- 自然处理累积数据，无需α调整
- 可纳入历史数据和专家意见
- 决策规则直观（概率解释）
- 适合复杂适应性设计

**缺点**：
- 先验选择可能引入主观性
- 计算复杂度高（需MCMC等）
- 监管接受度仍在发展中
- 频率学性质（如I类错误）需额外验证

## 4. 数据监查委员会（DMC）职责

### 4.1 DMC的定义

**数据监查委员会（Data Monitoring Committee, DMC）**，又称数据和安全监查委员会（DSMB），是独立于研究团队的专家小组，负责监测临床试验的安全性和有效性数据。

### 4.2 DMC的组成

| 角色 | 资质要求 |
|------|----------|
| **主席** | 临床试验设计专家，有DMC经验 |
| **统计学家** | 精通中期分析和序贯方法 |
| **临床专家** | 试验疾病领域的权威 |
| **伦理学家** | 熟悉受试者保护原则 |
| **患者代表** | 可选，提供患者视角 |

### 4.3 DMC的核心职责

```
┌─────────────────────────────────────────────────────────────┐
│                    DMC 核心职责                              │
├─────────────────────────────────────────────────────────────┤
│  1. 定期审查累积数据（安全性和有效性）                         │
│  2. 评估是否达到预定义的终止标准                              │
│  3. 向申办方提供继续/修改/终止试验的建议                      │
│  4. 审查方案偏离和严重不良事件                                │
│  5. 确保中期分析的保密性（避免操作偏倚）                      │
│  6. 撰写DMC报告并存档                                        │
└─────────────────────────────────────────────────────────────┘
```

### 4.4 DMC会议流程

1. **会前准备**
   - 接收盲态/非盲态数据报告
   - 统计师准备中期分析报告
   - 审阅安全性和有效性数据

2. **会议讨论**
   - 审查关键指标
   - 评估终止边界
   - 讨论潜在风险

3. **决策建议**
   - 继续试验
   - 修改方案（如样本量调整）
   - 提前终止（有效性/无效性/安全性）

4. **会后行动**
   - 向申办方提交建议
   - 记录决策依据
   - 保持信息保密

### 4.5 DMC章程（Charter）关键内容

- DMC组成和任命程序
- 会议频率和决策流程
- 终止标准定义
- 数据审查范围
- 保密要求
- 利益冲突声明

#### 5.3.2 贝叶斯中期分析

```r
# 模拟贝叶斯中期分析
simulate_bayes_interim <- function(n_total, n_interim, prior_mean, prior_var, 
                                    true_effect, sigma) {
  # 生成数据
  set.seed(123)
  data_interim <- rnorm(n_interim, true_effect, sigma)
  data_final <- rnorm(n_total, true_effect, sigma)
  
  # 先验
  prior_mean_val <- prior_mean
  prior_var_val <- prior_var
  
  # 中期后验
  post_var_interim <- 1 / (1/prior_var_val + n_interim/sigma^2)
  post_mean_interim <- post_var_interim * (prior_mean_val/prior_var_val + 
                                            sum(data_interim)/sigma^2)
  
  # 最终后验
  post_var_final <- 1 / (1/prior_var_val + n_total/sigma^2)
  post_mean_final <- post_var_final * (prior_mean_val/prior_var_val + 
                                        sum(data_final)/sigma^2)
  
  list(
    interim_post_mean = post_mean_interim,
    interim_post_var = post_var_interim,
    final_post_mean = post_mean_final,
    final_post_var = post_var_final
  )
}

# 运行模拟
result <- simulate_bayes_interim(
  n_total = 200,
  n_interim = 100,
  prior_mean = 0,
  prior_var = 100,
  true_effect = 0.5,
  sigma = 1
)

print(result)
```

#### 5.3.3 后验概率计算

```r
# 计算P(θ > δ_min | data)
posterior_prob <- function(post_mean, post_var, delta_min) {
  1 - pnorm(delta_min, mean = post_mean, sd = sqrt(post_var))
}

# 示例
prob_effective <- posterior_prob(
  post_mean = result$interim_post_mean,
  post_var = result$interim_post_var,
  delta_min = 0.2
)

cat("P(θ > 0.2 | 中期数据) =", round(prob_effective, 4), "\n")

# 决策
if (prob_effective > 0.975) {
  cat("建议：达到有效性标准，考虑提前终止\n")
} else if (prob_effective < 0.025) {
  cat("建议：达到无效性标准，考虑提前终止\n")
} else {
  cat("建议：继续试验\n")
}
```

## 6. 常见陷阱和注意事项

### 6.1 统计陷阱

| 陷阱 | 描述 | 避免方法 |
|------|------|----------|
| **α消耗不当** | 未正确控制总体I类错误 | 使用预定义的α消耗函数，严格遵循设计 |
| **信息时间错误** | 将日历时间误认为信息时间 | 基于事件数或有效样本量计算信息时间 |
| **边界误读** | 混淆Z值和p值 | 始终明确边界类型，使用软件验证 |
| **多重比较忽略** | 未考虑亚组分析的多重性 | 预定义亚组分析计划，调整α |
| **数据 peeking** | 非计划的中期查看数据 | 严格保密，仅DMC可接触非盲数据 |

### 6.2 操作陷阱

| 陷阱 | 描述 | 避免方法 |
|------|------|----------|
| **入组延迟** | 实际入组慢于计划，影响信息时间 | 设置灵活的α消耗函数，定期审查进度 |
| **失访率高** | 实际事件数少于预期 | 预设计算样本量时考虑失访率 |
| **边界交叉后继续** | 达到边界后仍继续入组 | 制定明确的终止流程，立即停止 |
| **DMC建议不执行** | 申办方忽视DMC建议 | 在协议中明确DMC建议的约束力 |
| **信息泄露** | 中期结果泄露导致偏倚 | 严格保密协议，限制数据访问 |

### 6.3 监管注意事项

- **预注册**：中期分析计划在试验注册时公开
- **协议修订**：任何中期分析计划的修改需经伦理委员会批准
- **透明度**：在最终报告中完整报告所有中期分析结果
- **监管沟通**：重大决策前与监管机构沟通（如FDA、EMA、NMPA）

## 附录：关键公式汇总

### A.1 组序贯边界

| 设计 | 边界公式 |
|------|----------|
| O'Brien-Fleming | $c_k = c \cdot \sqrt{k/K}$ |
| Pocock | $c_k = c$（常数） |
| Alpha Spending | $\alpha(t) = 2 - 2\Phi(z_{\alpha/2}/\sqrt{t})$ |

### A.2 后验分布（正态-正态）

$$
\text{Posterior: } \theta | y \sim N\left(\frac{\tau_0^{-2}\mu_0 + n\sigma^{-2}\bar{y}}{\tau_0^{-2} + n\sigma^{-2}}, \frac{1}{\tau_0^{-2} + n\sigma^{-2}}\right)
$$

### A.3 样本量（两样本均值）

$$
n = \frac{2\sigma^2(z_{1-\alpha} + z_{1-\beta})^2}{\Delta^2}
$$

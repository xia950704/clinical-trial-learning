## 二、模板使用规范

### 2.1 使用方式

**Obsidian内使用**：
1. 打开目标笔记
2. 在frontmatter中添加 `template: 模板名称`
3. 根据模板结构填充内容

**快速创建**：
```
_ template/模板-XXX.md → 复制 → 重命名 → 填充内容
```

### 2.2 Frontmatter标准

所有模板笔记必须包含：

```yaml
---
title: {{title}}
tags: [{{template_type}}]
status: template
template_type: {{type}}
created: {{date}}
updated: {{date}}
---
```

**特殊字段**：
- `status: template` — 模板文件专用状态
- `template_type` — 模板分类（个人档案/日记/玄学/健身/生活/求职/知识/理财/方法论/学习/系统）

### 2.3 内容结构标准

- 文件名：`模板-描述.md`
- 内容以 `# {{title}}` 开头
- 使用 `{{变量}}` 占位符（如 `{{date}}`、`{{name}}`）
- 以 `> 最后更新：YYYY-MM-DD` 结尾

## 三、新模板创建流程

### 3.1 需求分析

**确定模板类型**：
1. 明确模板用途（什么场景下使用）
2. 确定模板分类（对应 `template_type`）
3. 确定目标笔记类型（什么类型的笔记使用此模板）

### 3.2 结构设计

**标准结构**：
1. YAML Frontmatter（含 `template_type` 和 `status: template`）
2. 标题行（`# {{title}}`）
3. 模板说明（`> 说明文字`）
4. 内容结构（使用 `{{变量}}` 占位）
5. 使用示例（可选）
6. 最后更新日期

### 3.3 命名与存放

**命名规则**：
- 文件名：`模板-描述.md`
- 存放位置：`07-模板/模板-描述.md`

**编号规则**：
- 通用模板：`07-模板/` 目录
- 流程方法论模板：`04-方法学与工具/90-流程方法论/99-模板/` 目录

### 3.4 验证与索引

**验证步骤**：
1. 在测试笔记中应用模板
2. 检查frontmatter是否正确注入
3. 检查占位符是否可替换
4. 检查结构是否完整

**更新索引**：
- 在 `07-模板/00-模板索引.md` 中添加新模板条目

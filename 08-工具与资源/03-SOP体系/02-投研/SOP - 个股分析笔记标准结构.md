# SOP - 个股分析笔记标准结构（V1.0）

> 版本：V1.0 | 编制日期：2026-06-02

## 一、目标与交付物

**目标**：每只股票建立5个标准化MD文件，确保分析框架一致、信息完整、可追溯。

**交付物清单（每只股票5个文件）**：

| # | 文件名 | 内容 | 更新频率 |
|---|--------|------|:--------:|
| 1 | `{股票名}投资笔记.md` | 核心判断、一句话逻辑、估值状态、关键数据 | 每月 |
| 2 | `{股票名}-技术分析.md` | K线走势、均线、MACD、关键价格位 | 每周 |
| 3 | `{股票名}-事件时间线.md` | 重大事件按时间排序（BD/临床/财报/政策） | 实时 |
| 4 | `{股票名}-估值追踪.md` | PE/PB分位、历史估值区间、买入/卖出区间 | 每周 |
| 5 | `{股票名}-业务管线.md` | 核心产品/在研管线/竞品对比/商业模式 | 每季度 |

**Frontmatter标准**：

```yaml
---
title: {股票名}({CODE})投资笔记
code: {股票代码}
name: {股票中文名}
sector: {行业板块}
status: active  # active | 观察 | 已卖出
tags: [投研, 股票, {股票名}]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

## 二、标准工序（5个文件创建流程）

### 文件1：投资笔记（核心档案）

**结构**：

1. **核心判断**：一句话定性（如"创新药龙头，出海元年，GSK 125亿BD"）
2. **关键数据表**：
   - 创新药占比 / 在手订单 / PE / PB / 市值
3. **估值状态**：
   - 🟢 低估 / 🟡 合理 / ⚠️ 偏贵 / 🔴 高估
4. **投资逻辑**：
   - 核心驱动因素（3-5条）
   - 主要风险（3-5条）
5. **操作区间**（如适用）：
   - 买入区间 / 加仓区间 / 卖出区间

**踩过的坑**：
- ⚠️ 16只股票仅模板填充，9只深度分析，质量差距显著。→ 新股票创建时必须完成5文件全量结构，禁止只写投资笔记。

---

### 文件2：技术分析

**结构**：

1. **当前价格状态**：现价 / 52周高 / 52周低 / 距高比例
2. **均线系统**：MA5 / MA20 / MA60 / MA250 位置关系
3. **关键技术位**：支撑位 / 阻力位 / 成交密集区
4. **量价关系**：近期成交量变化、放量/缩量信号
5. **图表引用**：`!![`

**更新规则**：
- 每周更新一次价格数据
- 重大突破/跌破时即时更新

---

### 文件3：事件时间线

**结构**：

| 日期 | 事件类型 | 事件描述 | 影响评估 | 来源 |
|------|---------|---------|---------|------|
| YYYY-MM-DD | BD/临床/财报/政策 | 详细描述 | 正面/负面/中性 | 链接 |

**事件类型分类**：
- BD（商务拓展/授权合作）
- 临床（临床数据/申报/NDA/BLA）
- 财报（季报/年报/业绩指引）
- 政策（集采/医保/监管）
- 其他（管理层变动/诉讼/合作）

---

### 文件4：估值追踪

**结构**：

1. **当前估值**：PE(TTM) / PE(NTM) / PB / PS
2. **历史分位**：PE分位（近5年/近10年）
3. **估值区间**：
   - 历史最低PE / 历史最高PE / 当前PE
4. **操作价格**：
   - 买入区间（首笔/加仓）
   - 卖出区间
5. **估值变化日志**：
   - 每次更新记录价格变动及原因

**踩过的坑**：
- ⚠️ PE分位计算需使用AKShare价格分位替代，避免直接用PE数值分位。

---

### 文件5：业务管线

**结构**：

1. **核心产品**：
   - 产品名称 / 适应症 / 销售收入 / 占比 / 生命周期
2. **在研管线**：
   - 管线名称 / 适应症 / 临床阶段 / 预计上市时间 / 峰值销售预测
3. **竞品对比**：
   - 同类竞品列表 / 差异化优势 / 市场份额
4. **商业模式**：
   - 收入模式（自销/授权/CRO服务）
   - 成本结构
   - 现金流特征

## 三、批量创建流程

**适用场景**：新增股票时批量创建5个文件

**动作顺序**：
1. 确定股票代码、名称、行业板块
2. 创建投资笔记（文件1）—— 基础信息 + 核心判断
3. 创建事件时间线（文件3）—— 已有重大事件录入
4. 创建估值追踪（文件4）—— 当前估值 + 操作区间
5. 创建业务管线（文件5）—— 核心产品 + 在研管线
6. 创建技术分析（文件2）—— 初始价格状态
7. 建立文件间wikilink关联
8. 更新 `04-医药股票跟踪池.md` 添加新股票

**批量创建Prompt**：

```
为{股票名}({CODE})创建5个标准分析文件：
1. 投资笔记：核心判断+关键数据+估值状态+投资逻辑+操作区间
2. 技术分析：当前价格+均线+技术位+量价关系
3. 事件时间线：已有重大事件按时间排序
4. 估值追踪：当前PE/PB+历史分位+操作价格
5. 业务管线：核心产品+在研管线+竞品对比+商业模式

Frontmatter统一包含：code/name/sector/status/tags
文件间建立wikilink关联。
```

## 四、质量控制检查点

| QC编号 | 检查点 | 阈值 |
|:------:|:------|:----:|
| QC-1 | 5文件完整性 | 每只股票必须5个文件齐全 |
| QC-2 | Frontmatter完整性 | 必须含code/name/sector/status/tags |
| QC-3 | 文件间链接 | 投资笔记必须链接其他4个文件 |
| QC-4 | 价格数据时效性 | 技术分析/估值追踪每周更新 |
| QC-5 | 事件时间线完整性 | 重大事件24小时内录入 |

*本SOP基于01-股票投研目录中25只股票的实践总结。核心发现：9只深度分析（5文件齐全）vs 16只模板填充（仅投资笔记），质量差距显著。新股票必须5文件全量创建。*

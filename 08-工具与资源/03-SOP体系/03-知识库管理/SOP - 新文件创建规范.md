# SOP - 新文件创建规范

> **目的**：确保所有新创建的笔记文件按照SOP规范命名和存放，避免再次出现命名不规范、目录混乱的问题

---

## 一、创建前检查清单

在创建新文件前，必须执行以下检查：

### 1.1 搜索现有文件

```
搜索关键词：[项目编号]-[文件类型]-[中心名/描述]
```

**目的**：避免创建重复文件

**示例**：
- 创建274项目SAE报告前，搜索 `274 SAE`
- 创建合同笔记前，搜索 `274合同`

### 1.2 确认文件类型

| 文件类型 | 存放目录 | 命名前缀 |
|---------|---------|---------|
| 项目索引 | `01-索引/` | `项目名-` 或 `NN-` |
| 方案 | `02-方案/` | `项目名-` |
| 伦理 | `03-伦理/` | `项目名-` |
| eCRF | `04-eCRF/` | `项目名-` |
| 安全性 | `05-安全性/` | `项目名-` |
| 文献 | `06-文献/` | `项目名-` |
| 数据质量 | `07-数据质量/` | `项目名-` |
| 报告/知识库 | `08-知识库/` | `项目名-` |
| 汇总索引 | `09-索引/` | `项目名-` |

### 1.3 确认项目名称

| 项目 | 目录 | 前缀 |
|------|------|------|
| 274项目 | `09-百利/12-274项目/` | `274-` |
| 756项目 | `09-百利/13-756项目/` | `756-` |
| 240项目 | `09-百利/11-240项目/` | `240-` |
| 2000项目 | `09-百利/10-2000项目/` | `2000-` |

## 二、命名规范

### 2.1 标准格式

```
[项目编号]-[文件类型]-[版本/日期]-[描述].md
```

### 2.2 命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| ❌ 无空格 | 文件名中不使用空格 | `274-合同汇总.md` ✅ |
| ❌ 无特殊字符 | 不使用 `>>>`、`<<<`、`_` 等 | `274合同汇总.md` ✅ |
| ✅ 短横线连接 | 用 `-` 连接各部分 | `274-合同-安贞.md` ✅ |
| ✅ 版本优先 | 同一文件多个版本，按版本号命名 | `274-方案-V3.0.md` ✅ |
| ✅ 中文描述 | 描述部分使用中文 | `274-伦理-各中心汇总.md` ✅ |

### 2.3 命名示例

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| `274-合同汇总.md` | `274 合同汇总.md` |
| `274-方案-V3.0.md` | `274 临床试验方案 V3.0.md` |
| `274-伦理-各中心汇总.md` | `274各中心伦理批件汇总.md` |
| `274-SAE事件汇总报告.md` | `274 安全性事件汇总.md` |
| `274-数据质量问题清单.md` | `274 数据质量问题清单.md` |

### 2.4 特殊情况

| 情况 | 处理方式 | 示例 |
|------|---------|------|
| 通用文件 | 使用通用前缀 | `通用-合同模板.md` |
| 多项目对比 | 列出所有项目 | `274-756-方案对比.md` |
| 时间线记录 | 使用日期格式 | `274-时间线-20260603.md` |
| 报告类 | 使用报告前缀 | `274-报告-综合处理.md` |

## 三、frontmatter规范

### 3.1 必填字段

```yaml
---
title: [文件标题]
tags: [标签1, 标签2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: active|archived|draft
priority: P0|P1|P2
---
```

### 3.2 项目文件专用字段

```yaml
---
title: 274-合同-安贞主合同
tags: [274项目, 合同, 尼非卡兰]
created: 2026-06-03
updated: 2026-06-03
status: active
priority: P0
project: 274-核心RCT-尼非卡兰vs胺碘酮
version: V20211217
contract_number: HT2112357
center: 北京安贞医院
source: 移动硬盘/项目资料/274/274合同/安贞01/
---
```

### 3.3 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 文件标题（与文件名一致） |
| `tags` | ✅ | 标签（至少包含项目编号） |
| `created` | ✅ | 创建日期 |
| `updated` | ✅ | 最后更新日期 |
| `status` | ✅ | 状态：active/archived/draft |
| `priority` | ✅ | 优先级：P0/P1/P2 |
| `project` | ✅ | 所属项目 |
| `version` | ✅ | 版本号（如适用） |
| `source` | ✅ | 原始文件位置 |

## 五、创建流程

### 5.1 标准流程

```
1. 确定文件类型和存放目录
   └─ 根据文件内容选择正确目录
   
2. 搜索现有文件
   └─ 避免创建重复文件
   
3. 确定文件名
   └─ 按照命名规范生成文件名
   
4. 创建文件
   └─ 添加标准frontmatter
   └─ 添加文件内容
   
5. 建立链接
   └─ 在相关文件中添加链接
   └─ 更新项目索引
   
6. 记录创建
   └─ 在AI观察日志中记录
```

### 5.2 快速检查表

```
□ 文件名无空格、无特殊字符
□ 文件名使用短横线连接
□ 文件名包含项目编号
□ 文件放在正确目录
□ frontmatter完整
□ tags包含项目编号
□ 已搜索避免重复
□ 已建立相关链接
```

## 六、常见错误及修正

### 6.1 命名错误

| 错误 | 修正 |
|------|------|
| `274 合同汇总.md` | `274-合同汇总.md` |
| `274-合同_汇总.md` | `274-合同汇总.md` |
| `>>>274项目<<<-合同.md` | `274-合同汇总.md` |

### 6.2 目录错误

| 错误 | 修正 |
|------|------|
| 根目录创建 | 移至 `09-百利/12-274项目/01-索引/` |
| `09-百利/03-项目/` | 移至 `09-百利/12-274项目/` |
| `09-百利/274项目/` | 移至 `09-百利/12-274项目/` |

### 6.3 frontmatter错误

| 错误 | 修正 |
|------|------|
| 缺少title | 添加 `title: [文件名]` |
| 缺少tags | 添加 `tags: [274项目, ...]` |
| 缺少created | 添加 `created: YYYY-MM-DD` |
| 缺少status | 添加 `status: active` |

## 七、验证步骤

### 7.1 创建后立即验证

```
1. 检查文件名是否符合规范
2. 检查文件是否在正确目录
3. 检查frontmatter是否完整
4. 检查是否有重复文件
5. 检查链接是否正确
```

### 7.2 定期检查

| 检查项 | 频率 | 说明 |
|--------|------|------|
| 新文件命名检查 | 每次创建 | 创建后立即检查 |
| 目录结构检查 | 每周 | 检查文件是否在正确目录 |
| frontmatter完整性 | 每月 | 检查必填字段 |
| 重复文件扫描 | 每月 | 搜索相同标题的文件 |

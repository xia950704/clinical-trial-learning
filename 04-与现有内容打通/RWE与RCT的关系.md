# RWE 与 RCT 的关系

> **核心命题**：RCT 是疗效验证的"黄金标准"，RWE 是疗效泛化的"现实检验"。两者不是替代关系，而是互补共生的证据生态。
> 
> **一句话理解**：RCT 回答"在理想条件下药物是否有效"，RWE 回答"在真实世界中药物对谁、在什么情况下有效"。

---

## 一、真实世界证据（RWE）的定义和来源

### 1.1 核心定义

| 概念 | 定义 | 关键特征 |
|------|------|---------|
| **真实世界数据（RWD）** | 日常医疗实践中常规收集的数据 | 非研究驱动、多源异构、真实临床场景 |
| **真实世界证据（RWE）** | 对 RWD 进行分析后生成的关于药物效果和价值的临床证据 | 需经过严谨方法学处理、可支持监管决策 |

**FDA 定义**（2016 年《21st Century Cures Act》）：
> RWE 是"通过真实世界研究（RWS）生成的，关于药物的使用效果和价值的临床证据"，来源于对 RWD 的系统分析。

**关键区分**：RWD ≠ RWE。RWD 是原材料，RWE 是经过方法学"加工"后的证据产品。

### 1.2 RWD 的主要来源

| 数据类型 | 典型代表 | 优势 | 局限 |
|---------|---------|------|------|
| **电子健康记录（EHR）** | 医院 HIS 系统、EMR | 临床细节丰富、时变数据完整 | 数据质量参差、缺失值多 |
| **医保 claims 数据** | 国家医保、商业保险 | 覆盖广、随访长、结局客观 | 缺乏临床细节、编码误差 |
| **疾病登记系统** | 癌症登记、心血管登记 | 标准化收集、专业验证 | 覆盖病种有限、选择偏倚 |
| **可穿戴/远程监测** | Apple Watch、动态心电图 | 连续高频、患者自报告 | 依从性波动、数据噪声大 |
| **患者生成数据** | 健康 APP、患者论坛 | 患者视角、生活质量数据 | 自选择偏倚、数据标准化差 |
| **临床试验补充数据** | 扩展随访、患者注册 | 结合 RCT 严谨性+真实场景 | 成本较高、样本量有限 |

### 1.3 RWE 的监管应用

| 应用场景 | 说明 | 典型案例 |
|---------|------|---------|
| **支持新适应症的批准** | 在 RCT 不可行时提供证据 | 罕见病药物扩展适应症 |
| **上市后安全性监测** | 检测 RCT 中未发现的罕见 AE | 药物警戒、信号检测 |
| **标签更新** | 基于真实世界使用经验修改说明书 | 剂量调整、禁忌症更新 |
| **医保决策支持** | 为支付方提供效果-成本证据 | 药物经济学评估 |
| **加速审批通道** | 作为替代终点或补充证据 | FDA 突破性疗法认定 |

## 二、RCT 与 RWE 的互补关系

### 2.1 本质差异

| 维度 | RCT（随机对照试验） | RWE（真实世界证据） |
|------|-------------------|-------------------|
| **设计目的** | 验证疗效（内部有效性优先） | 评估效果（外部有效性优先） |
| **分配方式** | 随机化 | 自然分配（观察性） |
| **研究对象** | 严格纳入排除标准 | 广泛人群，接近临床实际 |
| **干预实施** | 标准化方案、严格依从性监控 | 临床常规实践、依从性变异 |
| **随访管理** | 主动随访、标准化评估 | 常规医疗接触、被动收集 |
| **混杂控制** | 随机化平衡已知+未知混杂 | 统计调整（PS、IPTW 等） |
| **结论外推** | 受限（严格入组条件） | 较强（真实人群代表性） |
| **主要偏倚** | 执行偏倚、失访偏倚 | 选择偏倚、信息偏倚、未测量混杂 |

### 2.2 互补矩阵：何时用 RCT？何时用 RWE？

```
                        内部有效性需求
                           ↑
                    高     │     高
              ┌────────────┼────────────┐
              │            │            │
              │  RCT       │  RCT + RWE │
              │  确证性试验│  桥接研究   │
              │            │            │
    外部      ├────────────┼────────────┤
    有效性    │            │            │
    需求      │  RWE       │  RWE       │
              │  探索性研究│  上市后监测 │
              │            │            │
              │  低        │     高     │
              └────────────┴────────────┘
                        外部有效性需求 →
```

**四象限解读**：

| 象限 | 场景 | 推荐策略 |
|------|------|---------|
| **高内部 + 高外部** | 确证性 III 期 → 上市后 | RCT + RWE 桥接（如扩展队列、真实世界外推分析） |
| **高内部 + 低外部** | 早期确证试验 | RCT 单独即可（如 Pivotal trial） |
| **低内部 + 高外部** | 上市后监测、比较效果研究 | RWE 单独（如登记研究、claims 分析） |
| **低内部 + 低外部** | 假设生成、信号发现 | RWE 探索性研究（如病例系列、生态学研究） |

### 2.3 互补的三种模式

#### 模式 1：RCT 先行 → RWE 外推（最经典）
- **流程**：RCT 确证疗效 → RWE 验证真实世界效果
- **适用**：新药上市后、适应症扩展
- **案例**：PD-1 抑制剂在 RCT 中确证有效 → RWE 在更广泛人群（包括 RCT 排除的肝肾功能不全患者）中验证效果

#### 模式 2：RWE 先行 → RCT 确证（加速路径）
- **流程**：RWE 发现信号/假设 → RCT 确证
- **适用**：老药新用、罕见病（RCT 入组困难）
- **案例**：SGLT2i 心衰获益最初来自 RWE 观察 → EMPA-REG OUTCOME RCT 确证

#### 模式 3：目标试验模拟（RWE 模拟 RCT）
- **流程**：用观察性数据"模拟"一个 RCT 的协议和分析
- **适用**：RCT 不可行（伦理/成本/时间）时替代
- **案例**：[[steroids-trial-emulation]]（详见第四节）

## 三、观察性研究设计要点

### 3.1 队列研究（Cohort Study）

**核心逻辑**：从暴露/治疗出发，前瞻或回顾性地追踪结局发生。

| 要素 | 要点 |
|------|------|
| **时间方向** | 前瞻性（gold standard）或回顾性（利用已有数据） |
| **暴露定义** | 明确、可重复、有时间锚点（如"首次处方日"） |
| **对照选择** | 未暴露/其他暴露，需与暴露组在基线可比 |
| **随访起点** | 必须与暴露时间一致（避免 immortal time bias） |
| **结局判定** | 预设标准、盲法评估（如可能） |

**关键偏倚及应对**：

| 偏倚 | 识别信号 | 应对策略 |
|------|---------|---------|
| ** immortal time bias** | 暴露组"存活"到用药的时间被错误归为暴露期 | 将时间划分为时变暴露，或使用新使用者设计（new-user design） |
| **选择偏倚** | 暴露组与对照组基线特征系统性差异 | PS 匹配/IPW、精确匹配 |
| **信息偏倚** | 结局评估标准不一致 | 标准化结局定义、盲法评估 |
| **失访偏倚** | 失访者特征与留存者不同 | IPW 处理删失、敏感性分析 |

**新使用者设计（New-User Design）** —— 观察性研究的"黄金准则"：
- 只纳入**首次**使用药物的患者
- 随访从**首次用药日**开始
- 排除已有用药史的患者（避免 prevalent user bias）
- 这是目标试验模拟的起点

### 3.2 病例对照研究（Case-Control Study）

**核心逻辑**：从结局出发，回溯暴露史。

| 要素 | 要点 |
|------|------|
| **适用场景** | 罕见结局（发病率 < 5%）、长潜伏期疾病 |
| **病例定义** | 明确的诊断标准、发病时间 |
| **对照选择** | 来自同一源人群、未发生结局、与病例在关键混杂上匹配 |
| **匹配因素** | 年龄、性别、入院时间等（避免过度匹配） |
| **效应指标** | OR（比值比），在罕见病假设下 ≈ RR |

**关键偏倚**：

| 偏倚 | 说明 | 应对 |
|------|------|------|
| **回忆偏倚** | 病例和对照回忆暴露史的准确度不同 | 使用客观记录（处方、检验）替代回忆 |
| **对照选择偏倚** | 对照不代表源人群 | 基于人群的对照、密度抽样 |
| **匹配过度** | 匹配了中介变量或暴露的后果 | 匹配前绘制 DAG，仅匹配混杂因子 |

### 3.3 交叉设计（Crossover Design）在观察性研究中的应用

**经典交叉设计**（RCT 中）：每个受试者依次接受所有处理，中间有洗脱期。

**观察性研究中的类比** —— 自身前后对照（Self-Controlled Designs）：

| 设计 | 原理 | 适用 | 局限 |
|------|------|------|------|
| **自身病例序列（SCCS）** | 同一病例的暴露期 vs 非暴露期比较 | 疫苗安全性、急性事件 | 要求事件不影响后续暴露 |
| **新使用者交叉分析** | 同一患者先后使用不同药物时的效果比较 | 序贯治疗的效果比较 | 需要足够的时间窗口、洗脱期 |
| **间断时间序列（ITS）** | 干预前后趋势的断点分析 | 政策/指南变更的影响 | 需要多个时间点的数据 |

**交叉设计的核心优势**：自身对照自动控制了所有**不随时间变化**的混杂（遗传、基线疾病严重程度等）。

## 四、因果推断方法详解

### 4.1 倒概率加权（IPTW / IPW）

**核心思想**：通过对每个个体赋予权重（1/PS 或 1/(1-PS)），构造一个"伪人群"，其中暴露与协变量独立，模拟随机化。

**权重公式**：
$$w_i = \frac{A_i}{e(X_i)} + \frac{1-A_i}{1-e(X_i)}$$

其中 $A_i$ 为暴露指示，$e(X_i) = P(A=1|X)$ 为倾向评分。

**实施步骤**：

1. **构建倾向评分模型**：Logistic 回归 / SuperLearner / BART
2. **计算权重**：稳定性权重（stable weights）可进一步缩放到样本均值
3. **权重截断**：截断极端值（如 1st/99th 分位数），避免个别权重主导
4. **加权分析**：在加权样本中运行结局模型（Cox / logistic / linear）
5. **平衡诊断**：检查加权后 SMD < 0.1（LOVE 图）

**优点**：
- 利用全部样本（不丢弃）
- 可处理多水平暴露
- 适合小样本（相比匹配）

**局限**：
- 极端权重导致估计不稳定
- 仅调整**测量**的混杂（未测量混杂仍存）
- 依赖倾向评分模型正确设定

**与已有笔记关联**：[[1076-ipw-weighting]]、[[1090-ps-weight-diagnostics]]、[[1087-msm-iptw]]

### 4.2 G-Computation（G 计算 / G-公式）

**核心思想**：基于"结构嵌套模型"的逆向思维——先建立结局对暴露和协变量的完整模型，然后**模拟**每个个体在每种暴露策略下的潜在结局，最后取平均得到因果效应。

**算法步骤**：

1. **拟合结局模型**：$E[Y|A,X]$（可用任意模型：Cox、logistic、BART、SuperLearner）
2. **预测潜在结局**：对每个个体 $i$，设定 $A=1$ 预测 $\hat{Y}_i(1)$，设定 $A=0$ 预测 $\hat{Y}_i(0)$
3. **计算个体效应**：$\hat{\tau}_i = \hat{Y}_i(1) - \hat{Y}_i(0)$
4. **平均效应**：$\hat{\tau} = \frac{1}{n}\sum \hat{\tau}_i$
5. **Bootstrap 求 CI**：重复抽样估计标准误

**时变暴露的 G-公式**（更复杂）：
- 需要模拟整个时变过程：$L_0 \to A_0 \to L_1 \to A_1 \to \dots \to Y$
- 对每个时间步拟合条件模型，然后蒙特卡洛模拟

**优点**：
- 利用全部数据、模型灵活
- 可估计**复杂动态策略**（如"出现低氧后 6 天给药"）
- 自然处理中介和交互

**局限**：
- 完全依赖模型正确设定（无双重稳健性）
- 时变版本计算复杂
- 高维协变量时模型可能过拟合

**与已有笔记关联**：[[IIT分析方法论与实战指南]] 中 Step 3 的方法对比表，以及 `cjyarnell/TargetTrial_ThresholdsForIMV` 仓库的 G 计算实践。

### 4.3 BART（Bayesian Additive Regression Trees）

**核心思想**：BART 是一种非参数贝叶斯回归方法，用许多回归树的和来拟合复杂函数关系，天然适合因果推断中的**潜在结局建模**。

**模型形式**：
$$Y = \sum_{j=1}^{m} g(X; T_j, M_j) + \epsilon, \quad \epsilon \sim N(0, \sigma^2)$$

其中每个 $g(\cdot)$ 是一棵回归树，$T_j$ 是树结构，$M_j$ 是叶节点参数。

**因果推断中的应用**：

| 用途 | 方法 |
|------|------|
| **倾向评分估计** | BART 回归 → 预测 $P(A=1|X)$ |
| **结局建模** | BART 分别拟合 $Y|A=1,X$ 和 $Y|A=0,X$ → 预测潜在结局 |
| **直接因果估计** | BART 将 $A$ 作为协变量纳入 → 从后验分布直接提取 CATE/ATE |
| **异质性分析** | BART 的叶节点天然提供个体化效应估计（CATE） |

**优点**：
- **非参数**：自动捕捉非线性、高阶交互
- **正则化**：树的数量和深度有先验约束，避免过拟合
- **不确定性量化**：天然提供 credible interval
- **双重稳健兼容**：可与 IPW 结合（DRL, Doubly Robust Learning）

**局限**：
- 计算量大（MCMC 采样）
- 解释性弱于参数模型
- 需要调参（树的数量、先验参数）

**R 实现**：`bartMachine`、`dbarts` 包
**Python 实现**：`PyBART`、`bartpy`

**与已有笔记关联**：`cjyarnell/TargetTrial_ThresholdsForIMV` 使用 BART 进行复杂结局预测；[[IIT分析方法论与实战指南]] 方法选择速查表中将 BART 列为"复杂预测/高维时变"的首选。

### 4.4 三种方法对比速查

| 维度 | IPTW | G-Computation | BART |
|------|------|--------------|------|
| **核心机制** | 加权创造伪随机化 | 模型预测潜在结局 | 非参数贝叶斯树和 |
| **模型依赖** | 仅 PS 模型 | 仅结局模型 | 无显式模型假设 |
| **双重稳健性** | ❌ | ❌ | 可与 IPW 结合实现 |
| **时变暴露** | MSM+IPTW | G-formula（天然支持） | 可扩展 |
| **计算复杂度** | 低 | 中 | 高 |
| **小样本表现** | 好（截断后） | 中（可能过拟合） | 好（正则化先验） |
| **异质性估计** | 需交互项 | 自然支持 | 天然支持（CATE） |
| **推荐场景** | 简单 A vs B、快速分析 | 动态策略、复杂干预 | 高维协变量、个体化效应 |

### 4.5 方法选择决策树

```
                    你的研究问题？
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
    A vs B 比较      动态/时变策略     个体化效应/
    (基线暴露)       (时变暴露)        异质性分析
        │                 │                 │
   ┌────┴────┐      LMTP /         BART /
   │         │      G-formula      因果森林
  n 大      n 小
   │         │
PS 匹配    IPW
   │         │
   └────┬────┘
        │
   验证平衡 → 结局模型 → 敏感性分析
```

## 五、与 steroids-trial-emulation 的关联

### 5.1 仓库概览

| 项目 | `kathoffman/steroids-trial-emulation` |
|------|--------------------------------------|
| **星级** | ⭐73 |
| **核心方法** | LMTP（Longitudinal Marginal Structural Model）+ SDR（Sequential Double Robustness） |
| **研究问题** | 激素治疗时长（出现低氧后几天开始给药）对 ICU 患者预后的影响 |
| **关键文件** | `code/analysis.R` —— 干预函数 + LMTP 估计 |
| **适用场景** | 时变暴露、动态治疗策略、离散时间点干预 |

### 5.2 目标试验模拟（Target Trial Emulation）的核心逻辑

**定义**：用观察性数据"重现"一个假设的 RCT 的协议、执行和分析流程。

**八个规格要素**（Hernán & Robins, 2016）：

| 要素 | 说明 | steroids 示例 |
|------|------|-------------|
| **eligibility** | 谁可以入组？ | ICU 患者、出现低氧（SpO₂<90%） |
| **treatment strategies** | 比较哪几种策略？ | 出现低氧后第 0/1/2/.../7 天开始激素治疗 |
| **assignment procedure** | 如何分配？ | 观察性 → 用 LMTP 模拟随机化 |
| **follow-up** | 随访多久？ | 从策略执行日起 30 天 |
| **outcome** | 主要终点？ | 30 天死亡率 |
| **causal contrast** | 因果对比？ | ITT（意向策略分析） |
| **analysis** | 分析方法？ | LMTP + SDR（非参数、机器学习） |
| **robustness** | 敏感性分析？ | 不同 k 值、不同 SL 候选、删失处理 |

### 5.3 steroids-trial-emulation 的方法学贡献

**1. 干预函数设计模式**（可复用）：

```r
# 干预函数：出现低氧后 trt 天给予治疗
int_treatment_after_event <- function(dat, trt) {
  trt_day <- parse_number(trt)
  earliest_day_to_check <- max(0, trt_day - 5)
  event_days_to_check <- paste0("E_", 
    str_pad(earliest_day_to_check:trt_day, width = 2, side = "left", pad = "0"))
  
  sum <- 0
  for(i in event_days_to_check){
    add <- ifelse(dat[[i]] == 1, 1, 0)
    sum <- sum + add
  }
  return(ifelse(sum > 0, 1, 0))
}
```

**核心思想**：将"动态策略"编码为可计算的函数，输入患者数据 → 输出是否遵循该策略的指示。这使得同一个 LMTP 框架可以评估**任意**动态规则。

**2. LMTP + SDR 的优势**：

- **LMTP**（Longitudinal Marginal Structural Model）：处理时变暴露和时变混杂的标准工具
- **SDR**（Sequential Double Robustness）：即使结局模型或暴露模型之一错误，估计仍一致
- **SuperLearner**：集成多个机器学习算法（BART、MARS、lasso、ridge、enet、mean），避免单一模型误设

### 5.4 与你的研究关联

| 你的研究 | steroids-trial-emulation 的启示 |
|---------|-------------------------------|
| **尼非卡兰 vs 胺碘酮复律成功率** | 如果考虑"重复给药时机"这一动态策略，可借鉴 LMTP 框架 |
| **时变暴露**（如用药时机、剂量调整） | 干预函数模式可直接复用：`int_treatment_after_event()` |
| **G-computation / BART** | steroids 仓库的 SuperLearner 候选包含 BART，可与你的 BART 分析打通 |
| **目标试验规格表** | 八个要素模板可直接用于你的 IIT 研究设计 |

### 5.5 从 steroids 到临床试验：RWE → RCT 的桥梁

**steroids-trial-emulation 的深层意义**：

1. **它证明了 RWE 可以达到接近 RCT 的因果推断质量**——通过清晰的试验协议 + 严谨的因果方法
2. **它为 RCT 不可行的场景提供替代方案**——如动态治疗策略的优化，在 RCT 中极难设计（需要多臂、多时间点）
3. **它为 RCT 设计提供参考**——观察性研究发现的"最佳时机"可以指导后续 RCT 的干预方案设计

**双向打通**：

```
观察性数据 → steroids-trial-emulation → LMTP 估计动态策略效果
     ↓                                        ↓
发现"出现低氧后第3天给药最优"    →    设计 RCT 验证该策略
     ↑                                        ↑
RCT 确证结果 ←── 外推至更广泛人群 ←── RWE 上市后监测
```

### 6.1 核心要点回顾

1. **RWE 不是 RCT 的替代品，而是补充**——RCT 保证内部有效性，RWE 保证外部有效性
2. **目标试验模拟是连接两者的桥梁**——用观察性数据模拟 RCT 协议，使 RWE 达到接近 RCT 的推断标准
3. **因果推断方法的选择取决于问题复杂度**——简单 A vs B → PS 匹配/IPW；时变暴露 → LMTP/G-formula；个体化效应 → BART
